# 技術架構

## 核心技術棧

| 層面 | 技術 | 版本 | 說明 |
|---|---|---|---|
| Framework | Next.js | 16+ | App Router |
| Language | TypeScript | 5+ | strict mode |
| Styling | Tailwind CSS | 4 | + CSS variables |
| Animation | Framer Motion | 12 | 元件動畫主力 |
| Smooth Scroll | Lenis | latest | 全站絲滑捲動 |
| Page Transition | View Transitions API | 原生 | 頁面 morph |
| Fuzzy Search | Fuse.js | latest | Cmd+K 全站搜尋 |
| 3D / Graph | react-force-graph-2d | latest | Landing hero 縮影 |
| Geometry | d3-delaunay | latest | Voronoi / Delaunay 視覺 |
| Map | Mapbox GL JS | latest | 世界地圖 zoom |
| State | Zustand | 5 | 輕量 global state |
| Icons | lucide-react | latest | 通用 icon |

## 資料策略

**全程 static data**：
- 所有 entity 資料以 TypeScript const 或 JSON 形式存放
- 無後端、無資料庫、無 API（除了 Next.js 路由本身）
- 內容更新 = 改檔案 → push → Vercel auto deploy

**為什麼不用 CMS / Notion API**：
- 內容寫完不常改
- static 載入最快、SEO 最好
- 0 維護成本、0 月費

## 內容生成流程

```
AI 草稿 (Claude API)
   ↓
人工校對正確性
   ↓
寫入 /content/{type}/{id}.md (MDX)
   ↓
Build-time 解析 → 生成 static pages
```

## 套件使用原則

| 功能 | 套件 | 原因 |
|---|---|---|
| 滑順 scroll | Lenis | 比 Locomotive 輕量、API 簡單 |
| 動畫 | Framer Motion | 不引入 GSAP（避免授權 + 體積） |
| 頁面切換 | View Transitions API | 原生支援、不裝額外套件 |
| 搜尋 | Fuse.js | 純前端 fuzzy match、無需後端 |
| 圖形 | react-force-graph-2d + d3-delaunay | 2D 為主、避免 Three.js 重量 |
| 地圖 | Mapbox GL | 比 Google Maps 視覺自由度高 |
| Icons | lucide-react | 線條風、符合視覺語言 |

> **原則**：能用 native web API 就不用套件、能用輕量套件就不用重量套件。
