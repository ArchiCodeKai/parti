# 資料模型設計

## Entity 統一基類

```typescript
type EntityType =
  | "architect"
  | "building"
  | "movement"
  | "school"
  | "firm"
  | "book"
  | "exhibition"
  | "award"
  | "theorist";

interface BaseEntity {
  id: string;
  type: EntityType;
  name: { zh: string; en: string; native?: string };
  wordCount: 200 | 300 | 500 | 800 | 1000;
  bodyText: string;             // MDX content
  relatedIds: string[];         // 80% auto + 20% manual
  imageUrl?: string;            // optional SVG line drawing
  tags: string[];               // 自由標籤
}
```

## Architect（人物）

| 欄位 | 型別 | 說明 |
|---|---|---|
| ...BaseEntity | | |
| lifespan | [number, number] | 生卒年 |
| nationality | string[] | 國籍（可多個） |
| movements | Movement[] | 多時期多流派 |
| buildings | string[] | 代表作 foreign key |
| firms | string[] | 屬於哪些事務所 |
| schools | { id, role, period }[] | 在哪受教、在哪任教 |
| influences | { from: string[], to: string[] } | 師生 / 影響 |
| books | string[] | 著作 |
| awards | string[] | 獎項 |
| isPritzker | boolean | Pritzker 得主標記 |

### movements 子型別（多時期格式）

```typescript
interface Movement {
  id: string;             // movement entity id
  period: [number, number];
  weight: "primary" | "secondary";
}
```

範例（Le Corbusier）：
```typescript
movements: [
  { id: "purism",              period: [1918, 1928], weight: "primary" },
  { id: "international_style", period: [1925, 1945], weight: "primary" },
  { id: "brutalism",           period: [1947, 1965], weight: "primary" },
]
```

## relatedIds 自動生成規則

每個 entity 的 `relatedIds` 80% 由系統自動匹配：

| 規則 | 適用 | 範例 |
|---|---|---|
| 同流派（任一時期） | architect ↔ architect | 同屬 brutalism |
| 同事務所 | architect ↔ architect | 同為 SOM 合夥人 |
| 同學派出身 | architect ↔ architect | 都來自包浩斯 |
| 師生關係 | architect ↔ architect | Mies → Johnson |
| 同建築物關聯 | architect ↔ building | Le Corbusier ↔ 廊香 |
| 同時代 ±10 年 | * ↔ * | 1950–1970 同時代 |
| 同國籍 | architect ↔ architect | 都是日本人 |
| 流派代表作 | movement ↔ building | brutalism ↔ Marseille Unité |

20% 由人工標註（特殊關聯、影響、對立等）。
