# 字數規範

## 為什麼要嚴格規範字數

- 視覺密度一致（馬賽克牆每張卡看起來資訊量近似）
- 用戶閱讀預期一致（不會某條目 100 字、某條目 5000 字）
- 卡片大小映射（字數 = 重要性 = 卡片 size）

## 寫作 tone 規範

### 統一 voice

- 第三人稱、客觀
- 不用「他被稱為...」（被動句）→ 用「業界稱他為...」
- 不過度抒情、不浪漫化
- 保留專業術語（不過度白話）

### 段落結構（500 字模板）

```
[段 1：身份定位 / 80 字]
姓名 + 國籍 + 生卒 + 一句話定位

[段 2：時代背景 / 100 字]
他在哪個年代、為什麼這個年代會出他

[段 3：核心貢獻 / 200 字]
主要設計理念、流派定位、代表作

[段 4：影響與評價 / 120 字]
影響了誰、被誰反對、後世如何評價
```

### 段落結構（800 字大師模板）

```
[段 1：身份定位 / 100 字]
[段 2：早期教育與背景 / 150 字]
[段 3：核心理念與宣言 / 200 字]
[段 4：代表作分析 / 200 字]
[段 5：流派貢獻與時代位置 / 100 字]
[段 6：晚年 / 評價 / 50 字]
```

## AI 生成 + 人工校對流程

### Step 1：AI 草稿（Claude API）

```
Prompt 模板：
請以第三人稱客觀 tone、繁體中文撰寫一段 [N] 字的建築師介紹。

人物：[名字]
類型：必選大師 / Pritzker / 普通
字數規範：[N] 字
段落結構：見字數模板

要求：
- 不浪漫化、不過度抒情
- 保留專業術語
- 包含：身份、時代、貢獻、影響
- 提及該人物的關鍵代表作 1–3 件
- 不出現「他被稱為...」這種被動贅句
```

### Step 2：人工校對

校對重點：
- [ ] 生卒年正確
- [ ] 流派標記正確
- [ ] 代表作年份正確
- [ ] 不出現「常識錯誤」（國籍弄反、師生關係錯）

### Step 3：交叉驗證

對照來源：
- Wikipedia（基本事實）
- 普利茲克官網（Pritzker 得主）
- 阮慶岳老師的著作（華人建築師）
- 漢寶德老師的著作（台灣戰後）

## 不允許的內容

- ✘ 政治評論（除非歷史事實必要、如 Albert Speer）
- ✘ 八卦 / 私人事件（除非歷史影響重大、如 Mies / 家族）
- ✘ 商業推薦（不導購事務所、不推薦造訪某建築）
- ✘ 投機性結論（「他的設計可能影響了...」這種沒根據的話）
