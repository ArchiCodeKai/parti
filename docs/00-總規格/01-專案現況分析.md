## 二、架構優點（值得保留的決策）

1. **依賴載入紀律好。** GSAP、Lenis、Three.js 全部 dynamic import 且有 cleanup，
   首頁 First Load JS 167kB、其餘頁 122–164kB，對這種視覺量級是健康的。
2. **資料引用完整性 100%。** architects ↔ buildings ↔ movements 三向引用經腳本驗證 0 斷鏈，
   這對後續內容擴充是可靠地基。
3. **設計系統 token 化完整。** `themes.css` 438 行涵蓋色彩、字級、間距、玻璃層、easing，
   landing 部分嚴格走 class；設計哲學（Glass Editorial / Hidden-until-Touched / N→幾何）有一致貫徹的意圖。
4. **核心演算法有品味。** `dispersion.ts` 的離散度計算（時代 30% / 流派 50% / 地理 20%、
   Jaccard 係數）簡潔正確；`nToGeometry()` 的 N→幾何映射是好的抽象。
5. **可及性有起步。** 卡片有 role/tabIndex/Enter、reduced-motion 全站處理、
   CustomCursor 在 touch 裝置自動退場、SiteHeader 有 aria-label。
6. **Fallingwater 模型有單元測試**（node:test），驗證建築原型比例 — 少見的用心。

## 三、架構缺點

1. **全站 client render 與「圖鑑」定位矛盾（最嚴重）。**
   17 個檔案 `"use client"`，三個 `[slug]` 詳情頁用 `useParams` 在瀏覽器取資料，
   build 結果為動態渲染（ƒ）、無 `generateStaticParams`、無 `generateMetadata`。
   後果：每一頁的 `<title>` 都一樣、搜尋引擎看不到條目內文、分享無條目資訊。
   百科型網站的長期價值就在搜尋長尾流量，這是結構性缺口，越晚改越貴
   （資料全靜態，本可 100% prerender）。
2. **樣式雙軌。** landing 用 themes.css class，子頁面用大量 inline style
   （單頁 500+ 行中 style 物件佔一半以上），同樣的 meta-label 樣式重複貼了十幾次。
   Tailwind v4 在管線中但 utility class 幾乎未使用。
3. **文件比程式碼多且互相矛盾。** 43+ 份規範文件已出現三處權威衝突
   （restrictions vs 現實、CLAUDE.md 進度 vs 現實、宣傳數字 vs 資料）。
   多 AI 協作模式下，過時文件會系統性帶偏每個新 session — 這是本次建立
   `docs/00-總規格/` 單一權威鏈的原因。
4. **幾何視覺目前是「裝飾」不是「資料」。** RelationGeometry 把 N 個點均勻擺在
   r=120 的圓上：三點定圓畫的圓與三點無幾何關係、Delaunay 是全連線、Voronoi 是虛線圈示意。
   幾何語言是本專案的核心賣點，示意版撐不起這個論述（d3-delaunay 已安裝但未使用）。

## 五、方向性誠實評估

**沒有致命的方向錯誤。** 定位（策展型建築圖鑑、幾何語言即論述、Glass Editorial 美學）
清晰且差異化，執行品質在 landing 已經接近投獎水準。但有三個必須直視的結構性風險：

1. **最大風險是內容產能，不是程式碼。**
   規格承諾 v1 200 條目 8 萬字、v2 600+ 建築 — 這是一個編輯部的工作量，不是一個人
   side project 的量。目前所有 bodyText 是 80–150 字 stub：網站殼再漂亮，
   評審或讀者點進任何詳情頁 10 秒就會看穿是 demo。
   **建議：v1 內容範圍正式鎖死在現有 29 / 68 / 27，把深度做滿（大師 800 字、
   Pritzker 500 字），放棄 v1 擴量。**「30 位 Pritzker 精選」本來就是 MVP-v1 的原始範圍，
   回到它。

2. **SEO 架構要在內容投資之前修好。**
   先寫 8 萬字再發現 Google 看不到，等於白寫。Phase 1（SSG 化）必須排在
   Phase 3（內容深化）前面 — 見開發路線圖。

3. **文件系統本身成了負債。**
   規範 43+ 檔對一人專案是過度工程，且已經在互相矛盾。本次交付的解法：
   `docs/00-總規格/` 四份文件 + 根目錄 `CLAUDE.md` 成為唯一權威鏈，
   舊文件降級為參考資料（權威順序見 CLAUDE.md）。今後規範只改權威鏈，不再增生。

另外兩點小提醒：

- 「Hidden-until-Touched」在 touch 裝置沒有 hover，目前個人頁 sidebar 靠 scroll 觸發
  算是有 fallback，但 RWD pass 時要逐一驗證每個隱藏元素在手機上有非 hover 的觸發路徑。
- `cursor: "none"` 散落在各元件 inline style（約 10 處），這其實是 CustomCursor 的
  全域責任，應收斂到 CSS 一處處理。
