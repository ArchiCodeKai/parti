## 二、架構決策記錄（ADR）

### AD-1 渲染策略：全站 SSG + Client Islands

**決策**：所有路由 build-time 預渲染。詳情頁改為 Server Component +
`generateStaticParams` + `generateMetadata`，互動部分抽成小型 client 元件
（TagBar、RelatedSidebar、CompareButton、Pullquote 等）。

**理由**：資料 100% 靜態（TS 模組），沒有任何 runtime 資料需求；
圖鑑的長期價值在搜尋長尾流量，必須讓每個條目有獨立 title/description 與
可被爬取的內文。

**否決的替代方案**：
- *現狀（全 client render）*：SEO 為零、分享卡無資訊，與產品定位直接矛盾。
- *SSR（runtime 渲染）*：沒有動態資料，白付伺服器成本。
- *`output: "export"` 全靜態輸出*：可行且是候選，但先不鎖死 —
  v2 條目級 OG image 可能需要 Vercel image API，屆時再決定。

### AD-2 資料層：TypeScript 模組即 CMS

**決策**：內容繼續放在 `src/lib/data/*.ts`（typed const array），
配 `scripts/validate-data.ts` 做 build 前引用完整性 + 字數檢查。

**理由**：200 條以下規模，TS 檔案給你免費的型別檢查、重構工具、git diff 可讀性；
一人專案內容進版本控制比進 CMS 合理。

**否決的替代方案**：
- *MDX 管線（docs 原規劃）*：等「長文 + 內嵌互動元件」需求真的出現再遷（v2 評估）；
  現在遷是為不存在的需求付重構成本。
- *Headless CMS / SQLite*：多一套基礎設施，違反「一個人能維護」約束。

### AD-3 動畫分工：Framer Motion 為主、GSAP 為特區

**決策**：元件級動畫（hover、présence、layout）一律 Framer Motion；
GSAP + ScrollTrigger + Lenis 只允許出現在 landing 的長卷軸敘事；兩者不得混用於同一元件。

**理由**：Framer Motion 與 React 生命週期整合好、宣告式；
ScrollTrigger 的 scrub/pin 是 landing 材質卷軸的剛需，Framer 的 useScroll 做不到同等控制。
劃清界線避免「同一效果兩套實作」。

**否決**：全 GSAP（React 整合成本高）；Locomotive Scroll（已定案 Lenis）。

### AD-4 3D 範圍：卡片級 vignette，不做全站 3D

**決策**：Three.js 只用於嵌入卡片的小型建築模型（現：Fallingwater），
必須 dynamic import、卸載時 dispose 材質/幾何/renderer、reduced-motion 給靜態版。
不引入 React Three Fiber。

**理由**：模型是「建築研究模型」的敘事道具，不是網站的骨架；
R3F 為單一 vignette 引入整套 reconciler 不划算。

### AD-5 地圖：純 SVG 等距柱狀投影

**決策**：維持現有純 SVG 實作（`/map`），移除 mapbox-gl 依賴。

**理由**：幾何座標場（經緯線 + 圖釘）與「幾何即論述」哲學一致；
零外部依賴、零 token 管理、離線可跑。寫實圖磚反而違反視覺規範。

**否決**：Mapbox GL（~200kB、需 access token、瓦片美學衝突）。
四層 zoom 敘事（流派發起地→城市）留 v2，用 SVG viewBox 動畫實現，仍不需要 Mapbox。

### AD-6 幾何視覺：從示意升級為真幾何

**決策**：`RelationGeometry` 升級 —
- N=3：以三點外心公式畫**真正的外接圓**（三點位置由資料決定，見下）。
- N=4–6：`d3-delaunay` 算真 Delaunay 三角網。
- N=7–12：`d3-delaunay` 的 `.voronoi()` 算真細胞。
- 點位規則（寫死、確定性）：角度 = 出生年在群體中的排序均分 360°，
  半徑 = 基準 120 × (1 ± 離散度貢獻)。禁止 Math.random。
- N≥13 力導向圖：v1 不實作（資料規模到不了），移除 react-force-graph-2d。

**理由**：幾何語言是核心賣點，假幾何撐不起論述；d3-delaunay 已在依賴中。

### AD-7 搜尋：Fuse.js 統一索引

**決策**：建 `src/lib/search/index.ts`，把三類 entity 映射成統一
`SearchDoc { id, type, route, nameZh, nameEn, tags, body }` 陣列餵 Fuse。

**否決**：Algolia（外部服務）、Pagefind（build 步驟複雜化）— 200 條目規模用不到。

### AD-8 樣式權威：themes.css tokens + component classes

**決策**：
- `src/app/themes.css` 是唯一 design token 與元件樣式來源。
- 重複 ≥ 2 次的樣式模式必須抽成 class（如 `.meta-label`、`.page-shell`）。
- inline style 只允許動態計算值（如 timeline 的 `left/width` 百分比）。
- 禁止硬編 hex / hsla —— 一律 `var(--token)`；流派色唯一來源是
  `movements.ts` 的 `colorTheme` 欄位。
- Tailwind v4 保留在管線（移除的收益不值得動 build 設定），但不強制使用；
  新程式碼優先用 component class。
- `src/design-reference/` 全目錄是唯讀歷史參考，不進 build（webpack null-loader），不同步維護。

### AD-9 狀態管理：Zustand 最小化

**決策**：全域狀態只有兩個 store（compare persist、cmdk open）。
訂閱一律走 selector（`useCompareStore((s) => s.ids)`），禁止整 store 解構。
Compare 分享改用 URL param 是 v2 範圍。

### AD-10 部署：Vercel（預設）

全站可 prerender，任何靜態託管皆可；預設 Vercel 以保留 v2 OG image API 的選項。

## 三、資料模型

### 3.1 現行三型別（`src/types/entity.ts`）

`BaseEntity`（id/type/name{zh,en,native}/wordCount/bodyText/tags/relatedIds/importance）
+ `Architect`（lifespan/nationality/movements[MovementPeriod]/buildings/awards/isPritzker…）
+ `Building`（architect/year{completed…}/location{lat,lng}/buildingType…）
+ `Movement`（era/originLocation/coreArchitects/keyBuildings/colorTheme/opposing…）。

### 3.2 需要的模型修正（任務對應見路線圖）

1. `Entity` union 移除 `BaseEntity` 成員，改為
   `type Entity = Architect | Building | Movement`（discriminated union，
   `type` 欄位可正常 narrowing）。
2. 在世者卒年慣例：`lifespan: [1941, 2099]`，2099 為「在世」哨兵值。
   新增 `formatLifespan(lifespan): string` helper 收斂散落 4 處的 `>= 2099` 判斷。
3. importance 配額制（避免通膨）：5 = 全站至多 6 席（改變典範者）、
   4 = Pritzker 常態、3 = 一般、2/1 = 補充。L/M/S/XS 卡片大小由此導出。
4. 六種未實作 Entity（school/firm/book/exhibition/award/theorist）：
   型別**先不定義**，architects 資料中的 `firms?/schools?/books?` 欄位視為
   前向引用（validate-data 不檢查其目標存在），v2 實作時再補。

### 3.3 資料不變量（validate-data 強制）

- 所有 `Architect.buildings[]` / `Building.architect` / `Movement.coreArchitects[]` /
  `Movement.keyBuildings[]` / `Building.movement` 引用必須存在。
- id 全域唯一（跨三檔）。
- `bodyText` 長度 = `wordCount` ± 15%（內容深化完成後啟用此規則）。
- `colorTheme` 必須是合法 CSS color。

## 五、品質門檻（每次改動的 Definition of Done）

```bash
npm run type-check   # 零錯誤
npm run lint         # 零警告
npm run build        # 零錯誤；檢查路由表：詳情頁必須是 ●(SSG) 而非 ƒ
npm test             # node:test 全綠（T0.3 接線後）
npm run validate     # 資料不變量全過（T0.3 新建後）
```

加上規範檢查：無新增硬編 hex、無 emoji 註解、繁體中文註解、雙引號、2 空格。
