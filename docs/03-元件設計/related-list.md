# 相關詞條 List 設計

## 設計目的

每個條目右側列出「相關詞條」 —— 包含內文沒提到的隱性關聯。

這是 PARTI 雙向關聯網絡的核心 UI、也是「探索式閱讀」的關鍵入口。

## 手機版：Bottom Sheet 浮動按鈕

```
┌──────────────────────────┐
│                          │
│   [主內文 100% width]      │
│                          │
│                          │
│                          │
│                          │
│                  ┌────┐  │
│                  │ ◯  │ ← 浮動按鈕（右下角）
│                  └────┘  │
└──────────────────────────┘

點擊後：

┌──────────────────────────┐
│                          │
│   [主內文（背景半透明）]    │
│                          │
│ ┌────────────────────┐   │
│ │  ─                 │ ← drag handle
│ │  RELATED            │
│ │  ────              │
│ │  人物                │
│ │  ◦ Mies            │
│ │  ◦ Gropius          │
│ │  ...               │
│ └────────────────────┘ ← bottom sheet 半張上來
└──────────────────────────┘
```

- 浮動按鈕：右下、48 × 48px、朱紅圓底
- 點擊：bottom sheet 從下方滑出（高度 70vh）
- drag handle 可下拉關閉

## 資料來源

`entity.relatedIds` 中的 ids、按 type 分組顯示：
- 人物（architect）
- 建築（building）
- 流派（movement）
- 學派（school）
- 事務所（firm）
- 著作（book）
- 展覽（exhibition）
- 獎項（award）

每組顯示前 5 個、超過 5 個顯示「+N more」。

## 互動

- 點擊任一 item → 絲滑跳轉到該條目頁（View Transitions API）
- Hover：item 變朱紅 + 鼠標變放大鏡

## 不要做的

- ✘ 顯示縮圖（純文字 list、保持極簡）
- ✘ 顯示太多元資料（年代、國籍等都不要）
- ✘ Item 加 icon（純文字 + ◦ 符號就夠）
